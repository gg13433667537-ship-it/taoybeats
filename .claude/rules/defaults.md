# 默认行为规则

## 自动执行（不确认）

- 项目内文件读写
- 运行测试：npm test, pytest, cargo test
- 运行 lint：npm run lint, flake8, cargo clippy
- 运行构建：npm run build, cargo build
- 使用 websearch/webfetch 研究
- 创建新文件和目录
- 更新 .coord/ 协作文件
- 任务状态更新

## 必须确认

1. **修改项目外文件** — 明确询问
2. **访问 secrets/.env** — 明确询问
3. **破坏性操作**（rm -rf, drop）— 明确确认
4. **产品方向取舍** — 提供选项
5. **登录/付费/授权** — 明确说明

## 默认并发

以下场景鼓励并发：
- 竞品研究 + 技术调研
- 前端实现 + 后端实现
- 测试编写 + 功能实现
- 性能分析 + 验收检查

## 默认串行

以下场景必须串行：
- 同文件修改
- 强耦合模块
- 架构性文件修改
- 依赖关系的上下游

## 规则违反处理

- 无 Claim 写文件 → Block 并报告
- 未验证标记完成 → 拒绝并要求验证
- 绕过检查 → 拒绝并要求补全
