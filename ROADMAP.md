# ROADMAP.md — AI Music 项目路线图

## 版本规划

### v0.1 — 奠基阶段（当前）

**目标**：建立项目框架，确认技术选型可工作

**里程碑**：
- [ ] GitHub 仓库 + Next.js 14 项目初始化
- [ ] shadcn/ui + Tailwind CSS 配置完成
- [ ] Prisma + PostgreSQL 数据库初始化
- [ ] NextAuth.js 认证流程打通（邮箱 + Google OAuth）
- [ ] 基础 UI 组件库就绪（Button / Input / Select / Card 等）

**可并发任务**：
- A1: Next.js 14 初始化 + TypeScript 配置
- A2: shadcn/ui + Tailwind 安装配置
- A3: Prisma Schema 设计（User / Song / Session 模型）
- A4: NextAuth.js 配置（邮箱 Provider + Google Provider）
- A5: 基础 CI/CD 流水线（GitHub Actions → Vercel）

---

### v0.2 — AI 生成核心（核心功能）

**目标**：用户能完整走通一次 AI 音乐生成流程

**里程碑**：
- [ ] AI Provider 抽象层（支持 Suno API / 通用接口）
- [ ] 歌曲生成表单页面（歌名 + 歌词 + 类型 + 情绪 + 乐器 + 备注）
- [ ] SSE 实时进度推送（后端 → 前端）
- [ ] 进度条 UI 组件 + 状态展示
- [ ] 生成结果展示页 + MP3 下载
- [ ] 游客模式（Cookie Session）

**可并发任务**：
- B1: AI Provider 抽象层（interface + SunoAdapter）
- B2: 歌曲生成表单 UI + 状态管理
- B3: SSE 进度推送 API Route
- B4: 进度条 UI 组件
- B5: 生成结果页 + 音频播放器
- B6: 下载功能（Signed URL）

**必须串行任务**：
- B1(B2) → B3: AI Provider 接口决定 SSE 的数据模型

## 延后池（v1.0+）

| 功能 | 优先级 | 理由 |
|------|--------|------|
| 声纹录制与克隆 | P0 延后 | 法律风险 + 后端依赖 |
| 完整支付系统 | P0 延后 | 合规复杂 |
| 歌词结构编辑器 | P1 | 复杂度高，MVP 可用纯文本 |
| 实时协作 | P2 | 优先级低 |
| 移动端 App | P2 | 先专注 Web |
| 内容审核 | P1 | 后期商业化需要 |

## 风险监控

| 风险 | 缓解方案 |
|------|----------|
| Suno API 封禁/变更 | 抽象 AI Provider 层，支持多 provider |
| 生成耗时长（10分钟） | 预估时间 + 后台生成 + 邮件通知 |
| 游客数据迁移失败 | 明确授权 + 迁移前备份 |
| 版权争议 | 服务条款 + 免责声明 |
